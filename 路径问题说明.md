# 路径中文字符问题解决方案

## 问题描述

错误信息：
```
Your project path contains non-ASCII characters.
Please move your project to a different directory.
```

## 原因

Android Gradle 插件不支持项目路径中包含非ASCII字符（如中文、日文等），这是 Android 构建工具的限制。

**当前问题路径：**
```
C:\任务列表\8、巡检脚本集合\ssl_apk_test
```
包含中文："任务列表" 和 "巡检脚本集合"

---

## 解决方案

### ✅ 方案1：移动项目到纯英文路径（强烈推荐）

#### 推荐的纯英文路径示例：
```
C:\Projects\ssl_apk_test
D:\AndroidProjects\ssl_apk_test
C:\work\ssl_apk_test
D:\dev\ssl_apk_test
C:\dev\android\ssl_apk_test
```

#### 操作步骤：
1. 在纯英文路径创建新文件夹，例如 `C:\Projects\`
2. 将 `ssl_apk_test` 整个文件夹复制到新位置
3. 使用新路径中的项目运行打包脚本

#### 优点：
- 符合 Android 官方规范
- 避免潜在的构建问题
- 兼容性好，不会遇到其他工具的路径问题

---

### ⚠️ 方案2：禁用路径检查（临时方案）

已自动在 `gradle.properties` 中添加以下配置：
```
android.overridePathCheck=true
```

#### 注意事项：
⚠️ **强烈不推荐生产环境使用**
⚠️ 可能导致以下问题：
1. 构建失败
2. 资源文件读取错误
3. Android Studio 无法正常打开项目
4. 其他 Android 工具不兼容

#### 仅适用于：
- 学习测试
- 快速验证
- 临时开发环境

## 建议的文件夹命名规范

### ✅ 推荐命名（纯英文，无空格）：
```
Projects/
├── android-projects/
│   ├── webview-app/
│   └── my-first-app/
├── ssl_apk_test/
└── demo-app/
```

### ❌ 避免命名（包含中文或特殊字符）：
```
任务列表/
8、巡检脚本集合/
my app/
demo@123/
```

---

## Android 路径限制总结

1. **不支持**：中文、日文、韩文等非ASCII字符
2. **不建议**：空格、特殊符号（@、#、$等）
3. **推荐**：纯英文、小写、连字符分隔

---

## 其他相关工具的路径要求

除了 Android Gradle，以下工具也有类似限制：

| 工具 | 是否支持中文路径 | 备注 |
|------|----------------|------|
| Android Studio | ❌ 不推荐 | 可能导致项目无法打开 |
| Gradle | ⚠️ 部分支持 | 可能构建失败 |
| React Native | ❌ 不支持 | 必须纯英文路径 |
| Flutter | ❌ 不支持 | 必须纯英文路径 |
| Ionic | ⚠️ 部分支持 | 可能出现问题 |

## 快速迁移步骤（推荐）

### Windows 操作：
1. 创建新文件夹：`C:\Projects\`
2. 复制 `ssl_apk_test` 到新文件夹
3. 在新位置打开项目
4. 更新工作区路径（如果使用 IDE）

### 批处理迁移脚本（可选）：
```cmd
@echo off
mkdir C:\Projects 2>nul
xcopy "ssl_apk_test" "C:\Projects\ssl_apk_test\" /E /I /Y
echo 项目已复制到: C:\Projects\ssl_apk_test
pause
```
